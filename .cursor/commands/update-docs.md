# Update docs

Follow `.cursor/skills/update-documentation/SKILL.md`.

1. Find last commit that touched docs (`git log` on `docs/`, README, AGENTS, `.cursor/` indexes).
2. Analyze code/config changes since that baseline.
3. Update stale engineering docs, indexes, and ONBOARDING; add ADR if architecture changed.
4. Verify and fix links across docs (Phase 5 — always).
5. Post summary of what was refreshed and link fixes.
